# 営業報告 Officeアドイン

Excelの「営業報告」シートを、タスクペインから **一覧／カンバン／集計** で操作できるアドインです。
新規入力はヘッダー右上の「＋新規入力」ボタンから行います（再読み込みボタンの左）。

**設定フラグ**: `app.js` 冒頭の `ENABLE_KANBAN_DND` を `true` にするとカンバンのドラッグ＆ドロップが有効になります（デフォルト `false`）。

## 画面構成

| 画面 | 機能 |
|---|---|
| 一覧 | 種別別グループ表示（**優先度列は左端**）。ワークフロー完了後の状態は「状態（m/d）」表示。状態フィルタは**複数選択可**。行クリック／右クリックで編集画面 |
| カンバン | 種別ごとのワークフローに沿った状態レーン（種別・状態フィルタは非表示）。DnDは `ENABLE_KANBAN_DND` フラグで制御（既定OFF） |
| 集計 | 保守状況／見積状況／売上予測 |
| ＋新規入力 | 起票情報のみ入力（対応状況は起票後に編集画面で入力） |

## 編集画面（ステージタブ式）

チケットの状態に応じたタブが活性化し、それ以外は 🔒 非活性になります。
**備考はタブの外にあり、どのステージでも編集できます。**

| タブ | 内容 | 完了時の遷移 |
|---|---|---|
| 起票 | 発生日・**起票者**・窓口・問合せ内容（新規入力時は担当者ではなく起票者を入力。担当者の初期値=起票者） | － |
| 対応中（保守・瑕疵・調整） | **対応担当者**（変更で担当者欄も更新）＋区分（保守対応のみ必須）＋対応状況 | 対応完了チェック→完了日を記録しチケット完了（開始日の右に完了日を表示） |
| 見積中（見積り） | **見積担当者**＋対応状況＋工数・価格（税抜）・**税込自動計算**・根拠 | 見積完了チェック→**見積完了日**を記録し確認中へ |
| 検討中（プリセールス） | **検討担当者**＋対応状況（検討・検証・提案作成など） | 検討完了チェック→**検討完了日**を記録し商談中へ |
| 商談中 | **窓口**＋商談状況 | 商談完了チェック→**商談完了日**を記録し確認中へ |
| 確認中 | 確認状況＋結果チェック | 「受注確定を完了にする」→**確認完了日**を記録・ステータスは確認中のまま受注タブ活性化／「失注」→確認完了日＋完了日を記録しチケット完了 |
| 受注 | 納品日・**計上日**・最終工数・最終価格（税込自動計算）・受注条件 | 完了チェック＋登録→ステータス「受注」・チケット完了 |

- 状態「新規」では対応状況欄は空のまま。**対応状況を記入して登録すると** 対応中／見積中／作成中へ遷移し、開始日（T列）を記録します。
- 完了したステージのタブは非活性となり、入力内容は参照のみ可能です。

## ワークフロー

- 保守対応・瑕疵対応・調整：`新規 → 対応中 → 完了`
- 見積り：`新規 → 見積中 → 確認中 → 受注 or 失注`
- プリセールス：`新規 → 検討中 → 商談中 → 確認中 → 受注 or 失注`
- どの状態からも「保留」に退避可能（カンバンDnD）

## 集計

- 集計期間は**期単位（10月〜翌9月、第37期=2025/10〜2026/09）**。◀▶ボタンで期を切り替えられます。
- **保守状況**: 保守対応＋瑕疵対応の月次推移を「発生」「完了」2系列のグラフで可視化。明細表に保守／瑕疵の内訳。
- **見積状況**: 見積り＋プリセールスを 新規→検討中→見積中→商談中→確認中→失注→受注 の順に件数・金額集計（受注は最終価格）。
- **受注状況**（旧: 売上予測）: 受注確定案件を**計上日ベース**で月別集計・グラフ化。計上日未入力の受注は警告表示。

### 顧客マスタシート（無ければ自動作成）

A:顧客コード（英字2〜4文字） B:取引先名 C:窓口 D:備考
案件番号は「顧客コード-通番」（例: KM-01）で自動採番します。

## デプロイ手順（GitHub Pages）

1. このフォルダ一式を `tools/addin/eigyo/` として GitHub Pages リポジトリに配置
2. `manifest.xml` をサイドロード、または Microsoft 365 管理センターから組織展開
3. Excelで対象ブックを開き、ホームタブの「営業報告」ボタンでタスクペインを表示

## デモモード

`index.html` をブラウザで直接開くと、Excelに接続せずサンプルデータで全機能を試せます（DEMOバッジ表示）。


## サイドメニュー（共通スライドメニュー）

ヘッダー左のハンバーカーボタン（≡）から、他アドインと共通のスライドメニューを開きます。
`tools/common/slide-menu.js` と `tools/common/menu.json` を遅延ロードして表示します。

- メニューには「操作（再読み込み）」と、共通の「ツール（カンバン → 営業報告 → バグ管理）」「リンク（使い方）」が並びます。
- カンバンの後ろに **営業報告**（id: `eigyo`）を追加した `menu.json` を同梱しています。共通ディレクトリ `tools/common/menu.json` をこの内容に差し替えてください（全アドインのメニューに一斉反映されます）。
- 現在表示中のアドイン（営業報告）はメニュー内で強調表示されます（`currentId: "eigyo"`）。

## この版の主な変更（rev_20260710_f）

### 詳細画面
- **削除ボタン**を追加。押下時に確認メッセージを表示し、OKで状態を「削除」に変更（一覧・カンバンから非表示になる）。
- 入力内容を変更した場合、画面外クリックでは閉じない（誤操作防止）。✕ボタンは確認ダイアログを表示。
- 一度登録した案件は**種別を変更不可**（グレー表示）。
- 登録するとタスクペインを閉じる。Excelの使用範囲が下方向に広がらないよう、読み書き範囲を使用範囲ベースに変更。

### 一覧・カンバン
- 状態が「削除」の案件は表示しない（状態フィルタの選択肢からも除外）。
- **左クリック**で該当のExcel行に移動し、その行を選択状態にする。
- 一覧はクリックした行をハイライト（排他・単一選択）。
- 詳細画面を開くには**右クリック**または**ダブルクリック**。
- 確認中で受注を選んだ案件も、状態表示は「確認中」に統一（受注バッジを撤去）。

### 保守の対応中
- 対応状況の下に「対応工数（人日）」欄を追加。

## この版の主な変更（rev_20260710_i）— 受託中ステージとWBS連携（案A）

- **受託中タブ**（2段階）:
  1. 状態=受注: 開始日（必須）・完了予定日を入力し「開始日を確定し、受託中にする」で状態=受託中へ。
  2. 状態=受託中: 「対応完了」チェックで完了日を記録しチケット完了。

## この版の主な変更（rev_20260710_j）— 受託スケジュール（ガントチャート）
- メインタブに**スケジュール**を追加（カンバンと集計の間）。中は「受託スケジュール（ガント）／受注状況グラフ」のサブタブ切替。
- **対象**: 状態が受注／受託中／完了の案件。バー=受託開始日(AH)〜納品日(N)（完了は完了日まで）。開始日・納品日が未確定の案件は斜線の仮表示。
- **操作**: バー両端ドラッグ=開始/終了の変更、本体ドラッグ=期間移動（AH列・N列へ即時書き戻し。完了案件は編集不可）。背景ドラッグで期間パン。細いバーは誤操作防止のため移動のみ（ズームインで両端編集可）。
- **表示**: 期(10月〜翌9月)単位で◀▶切替、ズーム12/6/3/1ヶ月（今日を中心に自動スクロール）、本日線＋今日チップ、開始/終了のm/d表示、タイトルはバー内に省略表示、納品日を▲m/dでプロット、凡例付き。
- カレンダーヘッダーは縦スクロールで上部固定（sticky）、案件ラベル列は横パンで左固定。
- 案件ラベルをクリックすると詳細画面を開きます。

## この版の主な変更（rev_20260710_k）

### 状態遷移
- 確認中タブ「受注確定を完了にする」→ 登録で状態が **受注** に。
- 受注タブ「この内容で登録し、受注確定する」→ 登録で状態が **受託中** に。

### 受託中タブ
- 開始日・完了予定日・納品日・計上日をすべて編集可能に（**完了予定日**は新列 AI）。
- 相関チェック: **開始日 ≦ 完了予定日 ≦ 納品日 ≦ 計上日**。違反時は該当項目を明示。

### フィルタ
- 種別・取引先を状態と同じ **複数選択** に統一。
- フィルタ条件を **Cookie に保存・復元**（次回起動時も維持）。

### スケジュール（ガント）

- 受注状況グラフのサブタブを撤去（受託スケジュールのみ）。
- 表示期間（ズーム・期）を **Cookie に保存**。

### 集計
- 保守状況: 発生・完了（棒）と対応工数（折れ線・右軸）を **1つの複合グラフ** に統合。工数表示チェックで折れ線ON/OFF。
- 見積一覧・受注状況の一覧: 右クリックで行選択、左クリックで詳細画面。

### その他
- 登録時に対象行の折り返しを無効化・行高さを固定し、Excelの行が縦に広がらないように。
