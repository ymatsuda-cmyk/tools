# シナリオ管理・バグ解消 作業フロー（シーケンス図）

## 📋 1. シナリオ作成状況管理のシーケンス

```mermaid
sequenceDiagram
    participant U as 作業者
    participant S as シナリオ管理
    participant B as バグ管理
    
    Note over U,S: ① 本日予定の設定
    U->>S: 本日実施予定に★をつける
    S-->>U: 本日予定として設定完了
    
    Note over U,S: ② 作業開始
    U->>S: 実施開始日を入力
    U->>S: 保存実行
    S-->>U: ステータス: 対応中
    
    Note over U,B: ③ バグ発生時の対応
    
    alt ③-1 新規バグ起票が必要
        U->>B: 新規バグを起票
        B-->>U: バグID発行
    end
    
    alt ③-2 軽微な課題の場合
        U->>S: 軽微な課題欄にバグID追加
        U->>S: 保存実行
        S-->>U: ステータス: 対応中（継続）
    else ③-3 完了阻害となる課題
        U->>S: 完了阻害となる課題欄にバグID追加
        U->>S: 保存実行
        S-->>U: ステータス: バグ保留
    end
    
    Note over U,S: ④⑤ 完了処理
    
    alt ④ 軽微な課題ありの完了
        U->>S: 実施完了日を入力
        U->>S: 保存実行
        S-->>U: ステータス: 完了（条件付き）
    else ⑤ 完全完了
        U->>S: 実施完了日を入力  
        U->>S: 保存実行
        S-->>U: ステータス: 完了
    end
```

## 🔍 2. バグ解消確認のシーケンス

```mermaid
sequenceDiagram
    participant U as 作業者
    participant B as バグ管理
    participant S as シナリオ管理
    
    Note over U,B: ① バグ完了確認の開始
    U->>B: 完了済みバグIDをクリック
    
    Note over B,S: ② 影響シナリオの表示
    B->>S: 該当バグIDで影響シナリオを検索
    S-->>B: 影響シナリオ一覧を返す
    B-->>U: 影響シナリオ一覧を表示
    
    Note over U,S: ③ 解消確認の実施
    loop 各シナリオについて
        U->>U: シナリオの状況確認
        alt 問題なし
            U->>B: 済にチェック
            B-->>U: 解消済みに設定
        else まだ問題あり
            U->>U: 確認継続
        end
    end
    
    Note over B,S: 自動ステータス更新
    alt 全影響シナリオが解消済み
        B->>S: バグ阻害解消を通知
        S->>S: 完了（条件付き）→完了 に更新
        S-->>B: ステータス更新完了
        B-->>U: バグ解消完了を通知
    end
```

## 📈 3. 総合ステータス遷移のシーケンス

```mermaid
sequenceDiagram
    participant U as 作業者
    participant S as シナリオ管理
    participant B as バグ管理
    
    Note over U,S: ステータス遷移の全体像
    
    S->>S: [新規] 状態で待機
    U->>S: ★をつける
    S->>S: [本日予定] に遷移
    
    U->>S: 開始日入力・保存
    S->>S: [対応中] に遷移
    
    alt バグなしで完了
        U->>S: 完了日入力・保存
        S->>S: [完了] に遷移
    else 軽微バグありで完了
        U->>S: 軽微課題追加・完了日入力
        S->>S: [完了（条件付き）] に遷移
        Note over B,S: バグ解消後
        B->>S: 軽微課題解消を通知
        S->>S: [完了] に遷移
    else 阻害バグで停止
        U->>S: 阻害課題追加・保存
        S->>S: [バグ保留] に遷移
        Note over B,S: バグ解消後
        B->>S: 阻害課題解消を通知
        S->>S: [対応中] に遷移
    end
```

## 🎯 シーケンス図の読み方

### 参加者
- **作業者**: シナリオ作成・バグ対応を行うユーザー
- **シナリオ管理**: シナリオの状況・ステータスを管理するシステム
- **バグ管理**: バグの起票・追跡・解消を管理するシステム

### 処理の流れ
- **→**: 同期的な処理・要求
- **-->>**: 非同期的な応答・通知
- **alt/else**: 条件分岐
- **loop**: 繰り返し処理
- **Note**: 処理段階の説明

### 重要なポイント
1. **バグ発生時の適切な分類**: 軽微 vs 阻害
2. **システム間連携**: バグ解消がシナリオステータスに自動反映
3. **効率的な確認フロー**: バグIDから影響シナリオを即座に特定
