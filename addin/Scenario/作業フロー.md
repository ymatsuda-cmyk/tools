# シナリオ管理・バグ解消 作業フロー（シーケンス図）

## 📋 1-1. シナリオ作成状況管理のシーケンス

```mermaid
sequenceDiagram
    participant U as 作業者
    participant S as シナリオ管理
    participant B as バグ管理
    
    Note over U,S: ① 本日予定の設定
    U->>S: 本日実施予定に★をつける（本日予定）
    
    Note over U,S: ② 作業開始
    U->>S: 実施開始日を入力（対応中）
    
    Note over U,B: ③ バグ発生時の対応
    U->>B: バグ起票
    
    U->>S: 軽微な課題欄にバグID追加（対応中）
    U->>S: 完了阻害となる課題欄にバグID追加（バグ保留）
    
    Note over U,S: ④⑤ 完了処理
    
    U->>S: 実施完了日を入力（完了（条件付き））
    U->>S: 実施完了日を入力（完了）

```

## 🔍 2. バグ解消確認の運用フロー

目的：解消したバグに対して、バグの解消確認を横展開にて行う。
対象：状態が確認、完了のバグ

```mermaid
sequenceDiagram
    participant U as シナリオ作成者
    participant S as シナリオ管理
    participant B as バグ管理
    
    Note over U,S: ① 影響シナリオの表示
    U->>S: バグIDをクリック
        S->>S: 該当バグIDで影響シナリオを検索
    S-->>S: 影響シナリオ一覧を返す
    S-->>U: 影響シナリオ一覧を表示
    
    Note over U,S: ② 解消確認の実施
    loop 各シナリオについて
        U->>U: バグ解消の確認
        alt 問題なし
            U->>S: 済にチェック（解消済み）
        else 問題あり
            U->>B: バグの起票
        end
    end
```

## 📈 1. シナリオ作成の運用フロー

```mermaid
sequenceDiagram
    participant U as 作業者
    participant S as シナリオ管理
    participant B as バグ管理
    
    Note over U,S: 本日実施予定（朝会）
    
    U->>S: ★をつける（本日予定）
    
    Note over U,S: シナリオ作成（開始時）
    U->>S: 開始日入力（対応中）
    
    Note over U,S: シナリオ作成（完了時／バグ発生時）
    alt 完了（バグなし）
        U->>S: 完了日入力（完了）
    else 軽微バグあり
        U->>S: 軽微課題の追加・完了日入力（完了（条件付き））
        Note over B,S: バグ解消後
        B->>S: バグ完了
U->>S: 解消を確認（完了）
    else 阻害バグあり
        U->>S: 阻害課題の追加（バグ保留）
        Note over B,S: バグ解消後
        B->>S: バグ完了
        U->>S: 解消を確認（対応中）→試験再開
    end
```

## 🎯 シーケンス図の読み方

### 参加者
- **作業者**: シナリオ作成・バグ対応を行うユーザー
- **シナリオ管理**: シナリオの状況・ステータスを管理するシステム
- **バグ管理**: バグの起票・追跡・解消を管理するシステム

### 処理の流れ
- **→**: 同期的な処理・要求
- **-->>**: 非同期的な応答・通知
- **alt/else**: 条件分岐
- **loop**: 繰り返し処理
- **Note**: 処理段階の説明

### 重要なポイント
1. **バグ発生時の適切な分類**: 軽微 vs 阻害
2. **システム間連携**: バグ解消がシナリオステータスに自動反映
3. **効率的な確認フロー**: バグIDから影響シナリオを即座に特定
