# 営業報告 Officeアドイン

Excelの「営業報告」シートを、タスクペインから **新規入力／一覧／カンバン／集計** で操作できるアドインです。
kanbanアドインと同じ構成（Office.js + GitHub Pages ホスティング）で作成しています。

## 機能

| タブ | 機能 |
|---|---|
| 新規入力 | 顧客マスタから取引先を選択 → 案件番号を自動採番（顧客コード-通番）。種別選択時にワークフローをプレビュー表示 |
| 一覧 | 種別（保守対応／瑕疵対応／見積り／プリセールス／調整）別のグループ表示。行クリックまたは**右クリック**で編集画面 |
| カンバン | 種別を選ぶと、その種別のワークフローに沿った状態レーンを表示。カードのドラッグ＆ドロップで状態遷移（不正な遷移はブロック）。カード右クリックで編集画面 |
| 集計 | 保守状況（月次推移グラフ）／見積状況（状態別 件数・金額）／売上予測（納品日ベース月別グラフ） |

### ワークフロー（種別別の状態遷移）

- 保守対応・瑕疵対応・調整：`新規 → 対応中 → 完了`
- 見積り：`新規 → 見積中 → 確認中 → 受注 or 失注`
- プリセールス：`新規 → 作成中 → 商談中 → 確認中 → 受注 or 失注`
- どの状態からも「保留」に退避でき、保留からは任意の工程に復帰できます
- 「受注」への遷移時は**見積金額と納品日が必須**（売上予測の元データになるため）
- 終端状態（完了／受注／失注）到達時は完了日を自動セット、受注区分（M列）も自動更新

## 対象シート仕様

### 営業報告シート（1行=1案件、1行目=ヘッダー）

| 列 | 項目 | 備考 |
|---|---|---|
| A | ID | 顧客コード-連番（例: KM-01）アドインが自動採番 |
| B | 取引先 | 顧客マスタの取引先名 |
| C | No | （旧様式互換・未使用） |
| D | 種別 | 保守対応／瑕疵対応／見積り／プリセールス／調整 |
| E | 状況 | ワークフロー状態（旧表記「検討中」「見積提出済み」等は自動読み替え） |
| F/G | 発生日／完了日 | 完了日は終端状態で自動セット |
| H/I/J | 担当者／窓口／優先度 | |
| K/L | 見積工数／見積金額 | 見積り・プリセールスのみ表示 |
| M | 受注区分 | 受注／失注（自動セット） |
| N | 納品日 | 売上予測の集計キー |
| O/P/Q/R | 内容／経緯／備考／要確認メモ | |

### 顧客マスタシート（無ければ初回起動時に自動作成）

| 列 | 項目 |
|---|---|
| A | 顧客コード（英字2〜4文字） |
| B | 取引先名 |
| C | 窓口 |
| D | 備考 |

初回作成時は営業報告シートの既存ID（KM-01等）から顧客コードを推定してシードします。

## デプロイ手順（GitHub Pages）

1. このフォルダ一式を `tools/addin/eigyo/` として GitHub Pages リポジトリに配置
   （manifest.xml内のURLは `https://ymatsuda-cmyk.github.io/tools/addin/eigyo/` 前提。変える場合はmanifest内の全URLを置換）
2. `manifest.xml` をサイドロード、または Microsoft 365 管理センターから組織展開
   - サイドロード（デスクトップ）：共有フォルダをアドインカタログに登録 → 挿入 > 個人用アドイン
   - 管理センター：設定 > 統合アプリ > カスタムアプリのアップロード
3. Excelで対象ブックを開き、ホームタブの「営業報告」ボタンでタスクペインを表示

## デモモード

`index.html` をブラウザで直接開くと、Excelに接続せずサンプルデータで全機能を試せます
（ヘッダーに DEMO バッジ表示）。UI修正時の動作確認に使えます。

## ファイル構成

```
eigyo/
├── manifest.xml     … アドインマニフェスト
├── index.html       … タスクペイン本体
├── app.js           … ロジック（Excel I/O・ワークフロー・カンバン・集計）
├── style.css        … スタイル（ローカル互換のため色はハードコード）
├── commands.html/js … リボンコマンド用
└── icons/           … アイコン（16/32/80px）
```
